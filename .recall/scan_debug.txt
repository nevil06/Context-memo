decisions:
  - decision: Use Gemini 1.5 Flash API for its free tier and capabilities.
    why: To provide a cost-effective solution for AI agent memory generation without requiring user credit card information.
    impact: The project relies on an external API for its core functionality.
  - decision: Structure the memory file in YAML format with a predefined schema.
    why: YAML is human-readable and easily parsable by both humans and machines, making it ideal for configuration and data exchange. The strict schema ensures consistency across different scans and agents.
    impact: All generated memory files adhere to this structure, simplifying parsing for AI agents.
  - decision: Implement a CLI interface using the 'commander' library.
    why: A CLI provides a convenient and scriptable way for developers to interact with the memory generation tool directly from their terminal.
    impact: The primary user interaction is through command-line arguments.
  - decision: Store user-specific configuration (like API keys) in a hidden directory in the user's home folder (~/.recall/config.json).
    why: This keeps sensitive information out of the project directory and ensures it's accessible across different projects.
    impact: Users need to configure the API key once per machine.
  - decision: Use a recursive directory scanner with skip patterns for efficiency.
    why: To avoid processing unnecessary files (like node_modules) and to limit the scope of the scan to relevant project files.
    impact: The scanning process is faster and more focused.
